📜 History of Java

Java is one of the world's most popular programming languages. It was designed to be simple, secure, portable and platform independent. Today Java powers millions of desktop applications, websites, enterprise software, Android applications and cloud services.

Origin of Java

Java was created by James Gosling along with Mike Sheridan and Patrick Naughton at Sun Microsystems in 1991. Initially the language was developed as part of the Green Project, whose goal was to build software for electronic devices such as televisions and home appliances.

Why was Java Created?

  • Create software that can run on different devices.
  • Provide better security than C and C++.
  • Reduce memory-related errors.
  • Support object-oriented programming.
  • Allow code to run on multiple operating systems without modification.

Why the Name "Java"?

The language was originally called Oak, named after an oak tree outside James Gosling's office. Since "Oak" was already trademarked, Sun Microsystems renamed it Java, inspired by Java coffee.

2006 Java became Open Source.

Present Java is used for Web, Android, Cloud, AI, Big Data and Enterprise applications.

Features that Made Java Popular

🏗️ Java Architecture

⚙️ How Java Works

Java follows a two-step execution process. First, the source code is compiled into bytecode. Then the Java Virtual Machine converts the bytecode into machine code for the operating system.

Step-by-Step Explanation

Step Description
1 Write the Java source code in a .java file.
2 The javac compiler converts source code into bytecode.
3 The generated bytecode is stored in a .class file.
4 The JVM loads the class using the Class Loader.
5 The Bytecode Verifier checks for illegal code and security issues.
6 The Execution Engine converts bytecode into machine code.
7 The operating system executes the machine code and displays the output.

Interpreter vs JIT Compiler

Interpreter JIT Compiler
Executes bytecode line by line. Compiles frequently used code into native machine code.
Slower execution. Much faster execution.
Uses less memory. Uses more memory but improves performance.

Why Java is Platform Independent?

Java programs are compiled into bytecode instead of machine code. Since every operating system has its own JVM, the same bytecode can execute without modification on Windows, Linux and macOS. This concept is known as Write Once, Run Anywhere (WORA).

Advantages of Java's Working Process

  • Platform Independent
  • Secure Execution
  • Automatic Memory Management
  • High Performance using JIT
  • Portable Applications
  • Reliable Execution
  • Easy Debugging
  • Supports Multiple Operating Systems

📦 Java Editions

Java is available in different editions to meet various development requirements. Each edition is designed for a specific type of application.

Edition Full Form Used For Examples
Java SE Standard Edition Desktop applications, console applications and core Java development. Calculator, Banking System, Inventory Software
Jakarta EE
(Java EE)		 Enterprise Edition Large-scale enterprise and web applications. Banking Portals, E-Commerce Websites, ERP Systems
Java ME Micro Edition Embedded systems and IoT devices. Smart TVs, Medical Devices, Industrial Equipment
Java Card Java Card Technology Applications on smart cards and SIM cards. ATM Cards, SIM Cards, Security Cards

Java SE (Standard Edition)

  • Foundation of Java programming.
  • Contains JVM, JRE and core libraries.
  • Supports OOP, Collections, Multithreading, JDBC, File Handling and Networking.
  • Used by beginners as well as professionals.

Jakarta EE (Formerly Java EE)

  • Built on Java SE.
  • Provides enterprise APIs.
  • Supports Servlets, JSP, EJB, JPA and Web Services.
  • Ideal for enterprise web applications.

Java ME

  • Designed for devices having limited memory.
  • Lightweight Java platform.
  • Mainly used in embedded systems.

Java Card

  • Runs Java applications inside smart cards.
  • Provides high security.
  • Used in banking and telecommunication industries.

🔑 Java Keywords

Keywords are reserved words that have predefined meanings in Java. They cannot be used as variable names, class names or method names.

Access Modifiers

public private protected

Data Types

byte short int long
float double char boolean

Object-Oriented Keywords

class interface extends implements
this super new instanceof

Control Flow Keywords

if else switch case default
for while do break continue
return goto*

* goto is reserved but not used in Java.

Exception Handling Keywords

try catch finally throw throws

Package Keywords

package import

Inheritance & Abstraction

abstract final native strictfp

Multithreading Keywords

synchronized volatile transient

Miscellaneous Keywords

void static const* enum
assert var

* const is reserved but not used in Java.

Important Notes

  • Java has 50 reserved keywords.
  • Keywords are case-sensitive.
  • You cannot use keywords as identifiers.
  • true, false and null are literals, not keywords.
  • goto and const are reserved for future use.

🛠 Java Development Tools

Java developers use a variety of tools for writing code, compiling, debugging, testing, building and deploying applications.

JDK

Java Development Kit contains the compiler, debugger, JRE and other development utilities.

JRE

Java Runtime Environment provides the libraries required to run Java applications.

JVM

The Java Virtual Machine executes Java bytecode and makes Java platform independent.

IntelliJ IDEA

Professional IDE developed by JetBrains. Popular for enterprise Java development.

Eclipse IDE

Open-source IDE widely used for Java, Spring Boot and enterprise projects.

Apache NetBeans

Official Apache IDE supporting Java, PHP and web development.

Visual Studio Code

Lightweight code editor with Java extensions and debugging support.

Maven

Automates project management, dependency handling and builds.

Gradle

Modern build automation tool known for faster builds.

Git

Version control system used to track changes in source code.

JUnit

Testing framework used to write and execute unit tests.

Apache Tomcat

Web server and servlet container used for Java web applications.
