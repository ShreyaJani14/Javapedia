M

Terms Defination
Main Method The starting point of a Java application: public static void main(String[] args).
Map An interface that stores data as key-value pairs.
MappedByteBuffer A buffer that maps a file directly into memory for fast I/O operations.
Marker Interface An interface with no methods used to indicate special behavior, such as Serializable.
Math Class Utility class providing mathematical functions like sqrt(), pow(), and abs().
Member Any field, method, constructor, or nested type declared in a class.
Member Method A method that belongs to a class or object.
Member Variable A variable declared inside a class but outside methods.
Memory Allocation The process of reserving memory for objects and variables.
Memory Leak Occurs when unused objects remain referenced and cannot be garbage collected.
Memory Management JVM's process of allocating and reclaiming memory.
Memory Pool A logical memory area managed by the JVM.
Memory-Mapped File A file mapped into memory for efficient reading and writing.
Memento Pattern A design pattern used to save and restore an object's state.
Message Queue A structure used for asynchronous communication between components.
MessageDigest A class used to create cryptographic hash values such as SHA-256.
Metadata Data describing other data, often provided through annotations.
Meta-Annotation An annotation applied to another annotation.
Method A block of code that performs a specific task.
Method Area JVM memory area storing class metadata and static variables.
Method Call Invocation of a method.
Method Chaining Calling multiple methods in a single statement.
Method Declaration Definition of a method's name, return type, and parameters.
Method Handle A typed reference to a method introduced in Java 7.
Method Hiding When a static method in a subclass hides a static method in the parent class.
Method Invocation The act of executing a method.
Method Local Variable A variable declared inside a method.
Method Overloading Multiple methods with the same name but different parameters.
Method Overriding Redefining a superclass method in a subclass.
Method Parameter Input value accepted by a method.
Method Reference Java 8 feature using :: syntax to refer to methods.
Method Signature Combination of method name and parameter list.
Microbenchmark A benchmark that measures performance of small code segments.
Microservice An architectural style where applications consist of small independent services.
Migration Moving applications or code to newer versions or platforms.
Mixin A design concept that adds reusable functionality to classes.
Mock Object A simulated object used during testing.
Mocking Creating fake implementations for testing purposes.
Model Represents application data and business logic.
Model Class A class that stores and manages data.
Model Object An instance representing application data.
Model-View-Controller (MVC) Design pattern separating data, UI, and control logic.
Modifier Keyword that changes access or behavior (public, private, static).
Module A named collection of packages introduced in Java 9.
Module Descriptor module-info.java file defining module dependencies.
Module Path Location where Java modules are stored.
Modular Programming Developing software using independent modules.
Monitor Synchronization mechanism controlling thread access to shared resources.
Monitor Lock Lock associated with an object for synchronization.
Monolithic Application Application deployed as a single unit.
Multicast Sending data to multiple recipients simultaneously.
MulticastSocket Java networking class supporting multicast communication.
Multi-Catch Feature allowing multiple exceptions in one catch block.
Multi-Module Project A project composed of several modules.
Multidimensional Array Array containing other arrays.
Multilevel Inheritance Inheritance chain involving multiple classes.
Multithreading Running multiple threads concurrently.
Mutable Class A class whose objects can be modified after creation.
Mutable Object An object whose state can change.
Mutable Collection A collection that allows modifications after creation.
Managed Bean A JavaBean managed by a framework.
Managed Thread A thread whose lifecycle is controlled by a container or framework.
Mapper A component that converts one object type to another.
Mapper Class Class responsible for data transformation.
Mapping Associating one object or value with another.
Manifest File MANIFEST.MF file containing metadata about a JAR.
Maven Repository Storage location for Maven dependencies.
Maven Dependency External library declared in Maven configuration.
Maven Plugin Component extending Maven build functionality.
Maven Project Java project managed using Maven.
Maven Lifecycle Sequence of build phases such as compile, test, and package.
Managed Resource Resource whose lifecycle is controlled by a framework.
Message Listener Component that receives and processes messages.
Message Producer Component that sends messages.
Message Consumer Component that receives messages from a queue or topic.
Middleware Software layer connecting applications and services.
Marshalling Converting an object into a transferable format such as XML or JSON.
Method Stub Placeholder method with incomplete implementation.
Method Body Block containing statements executed by a method.
Method Scope Region where a method's local variables are accessible.
Method Return Type Data type returned by a method.
Method Contract Expected behavior and obligations of a method.
Method Resolution Process of determining which method to invoke.
Method Dispatch Runtime mechanism selecting the correct overridden method.
Module System Java 9 feature supporting modular applications.
Memory Consistency Rules governing visibility of shared data between threads.
Memory Barrier Synchronization mechanism ensuring correct memory visibility.
